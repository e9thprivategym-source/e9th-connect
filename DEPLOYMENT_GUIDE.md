## 環境変数 (Environment Variables)

E9th Connect アプリケーションは、以下の環境変数に依存しています。Vercel や Supabase などのデプロイ環境にこれらの変数を設定してください。

| 環境変数名           | 説明                                                              | 設定例                                      |
| :------------------- | :---------------------------------------------------------------- | :------------------------------------------ |
| `VITE_APP_ID`        | アプリケーションのID (クライアントサイドで使用)                   | `e9th-connect-app`                          |
| `JWT_SECRET`         | JWT の署名に使用されるシークレットキー                            | `your_jwt_secret_key_here`                  |
| `DATABASE_URL`       | データベース接続文字列 (Supabase または Neon の接続文字列)        | `postgresql://user:password@host:port/db`   |
| `OAUTH_SERVER_URL`   | OAuth サーバーのURL (認証に使用)                                  | `https://oauth.example.com`                 |
| `OWNER_OPEN_ID`      | アプリケーションオーナーの Open ID                                | `owner_open_id_here`                        |
| `NODE_ENV`           | アプリケーションの実行環境 (`production` または `development`)    | `production`                                |
| `BUILT_IN_FORGE_API_URL` | Forge API の URL                                                  | `https://api.forge.example.com`             |
| `BUILT_IN_FORGE_API_KEY` | Forge API のキー                                                  | `your_forge_api_key_here`                   |
| `PORT`               | アプリケーションがリッスンするポート番号 (Vercel では自動設定)    | `3000`                                      |
| `VITE_APP_URL`       | アプリケーションの公開URL (クライアントサイドで使用)              | `https://e9th-connect.vercel.app`           |

## Vercel へのデプロイ手順

1. **Vercel アカウントの作成**: Vercel のウェブサイトにアクセスし、アカウントを作成します。
2. **Git リポジトリの連携**: E9th Connect のコードを GitHub, GitLab, または Bitbucket のリポジトリにプッシュします。Vercel ダッシュボードから新しいプロジェクトを作成し、該当する Git リポジトリを連携します。
3. **環境変数の設定**: Vercel プロジェクトの設定画面で、上記の「環境変数」セクションに記載されている変数を追加します。特に `DATABASE_URL` は Supabase または Neon から取得した接続文字列を設定してください。
4. **デプロイ**: 環境変数を設定後、Vercel は自動的にアプリケーションをデプロイします。手動でデプロイを開始することも可能です。

## Supabase/Neon データベースの設定手順

1. **Supabase/Neon アカウントの作成**: Supabase または Neon のウェブサイトにアクセスし、アカウントを作成します。
2. **新しいプロジェクトの作成**: ダッシュボードから新しいプロジェクトを作成し、データベースをプロビジョニングします。
3. **データベース接続文字列の取得**: 作成したデータベースの接続設定から、`DATABASE_URL` を取得します。このURLを Vercel の環境変数に設定します。
4. **マイグレーションの実行**: 後述の「データベースマイグレーション」セクションで出力される SQL ファイルを使用して、Supabase/Neon データベースにスキーマを適用します。


## データベースマイグレーション

PostgreSQL (Supabase/Neon) 用の初期スキーマ SQL ファイルが `drizzle/0000_cute_chronomancer.sql` に生成されています。

### 適用方法

1. Supabase または Neon の SQL エディタを開きます。
2. `drizzle/0000_cute_chronomancer.sql` の内容をコピーして貼り付けます。
3. 実行ボタンを押して、テーブルと列を作成します。

> **注意**: `--> statement-breakpoint` は Drizzle Kit 特有の区切り文字です。SQL エディタで直接実行する場合は、これらを削除するか、1つずつステートメントを実行してください。
